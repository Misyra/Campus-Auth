JSON 配置说明

支持的步骤类型

  • input - 输入文本
  • click - 点击元素
  • click_select - 点击并选择（点击元素后从弹出菜单中选择）
  • select - 选择下拉框
  • wait - 等待元素出现
  • wait_url - 等待URL匹配
  • goto - 页面导航（与顶层 url 自动导航互补，用于执行过程中切换页面，支持 wait_until 字段）
  • eval - 执行JS（支持返回值存储到变量）
  • screenshot - 保存截图
  • sleep - 等待指定时间
  • ocr - 验证码识别（ddddocr），截图识别图片文字并可选填入输入框
    • selector - 验证码图片/Canvas 选择器（必填）
    • target_selector - 验证码输入框选择器（可选，设置后自动填入识别结果）
    • store_as - 将识别结果存入变量（可选，配合 eval 步骤做二次处理）
    • char_range - 限定 OCR 识别的字符范围，提高准确度（可选）。数字 0~7 使用内置字符集，或传入自定义字符串如 "0123456789"
    • old - 使用旧版模型（可选，默认 false）

公共字段

  • frame - 目标 frame 名称、URL 片段或 CSS 选择器（字符串，不支持布尔值），用于 frameset/iframe 页面

顶层配置

  • reveal_hidden - 执行前强制显示所有隐藏输入框（布尔值，默认 false）。通常无需开启，每个步骤在普通操作失败后会自动降级到强制模式
  • step_delay - 步骤间休眠时间（秒，默认 0.5）
  • navigation_wait - 页面加载后额外等待时间（秒，默认 1）。适用于页面通过 AJAX 动态渲染表单的场景，等待时间不足会导致后续步骤找不到元素
  • success_condition - 成功条件变量名（可选，字符串）。声明后系统在步骤跑完后从运行时变量中读取该值做真值判定：真值→成功；假值→失败（按重试策略重试）。登录任务声明后跳过网络检测。配合 eval 步骤的 store_as 使用：在 eval 步骤里写 JS 判定逻辑存到变量，顶层用此字段引用变量名。单纯的校园网登录任务可以不设，系统会自动通过网络检测判断登录是否成功

可用变量

  • {{USERNAME}} / {{username}} - 用户名（系统变量 / 需 variables 映射）
  • {{PASSWORD}} / {{password}} - 密码（系统变量 / 需 variables 映射）
  • {{ISP}} / {{isp}} - 运营商后缀（系统变量 / 需 variables 映射）
  • {{LOGIN_URL}} / {{url}} - 认证地址（系统变量 / 需 variables 映射）

示例

{
  "variables": {"username": "{{USERNAME}}", "isp": "{{ISP}}"},
  "navigation_wait": 3,
  "reveal_hidden": false,
  "steps": [
    {
      "type": "input",
      "selector": "#username",
      "value": "{{username}}",
      "description": "输入用户名"
    }
  ]
}

成功判定示例（用 eval 步骤识别密码错误）

在 eval 步骤里写 JS 判定逻辑存到变量，顶层用 success_condition 引用变量名。声明后登录任务跳过网络检测：

{
  "name": "校园网登录",
  "steps": [
    {"id": "s1", "type": "goto", "url": "{{auth_url}}"},
    {"id": "s2", "type": "input", "selector": "#username", "value": "{{username}}"},
    {"id": "s3", "type": "input", "selector": "#password", "value": "{{password}}"},
    {"id": "s4", "type": "click", "selector": "#login"},
    {
      "id": "check_result",
      "type": "eval",
      "description": "页面无错误提示即视为成功",
      "script": "() => !document.body.innerText.includes('密码错误') && !document.body.innerText.includes('认证失败')",
      "store_as": "success_flag"
    }
  ],
  "success_condition": "success_flag"
}

OCR 验证码示例

普通图片验证码（识别后直接填入输入框）：

{
  "type": "ocr",
  "selector": "#captchaImg",
  "target_selector": "#captchaInput",
  "char_range": "0123456789"
}

数学运算验证码（识别图片 → 存变量 → eval 计算结果）：

[
  {
    "type": "ocr",
    "selector": "#captchaCanvas",
    "store_as": "captcha_expr",
    "char_range": "0123456789+-*/=xX÷",
    "old": true
  },
  {
    "type": "eval",
    "script": "() => { let expr = '{{captcha_expr}}'; /* 计算逻辑 */ }",
    "store_as": "captcha_result"
  }
]
