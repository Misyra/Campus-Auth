## 快速开始

### 启动

**Windows 用户**

双击运行 `start.exe`，程序会自动下载运行环境并启动 Web 控制台界面。如果没有自动打开，请手动访问 `http://127.0.0.1:50721`，或点击右下角托盘图标。

启动后填入账号密码等信息，点击"启动监控"即可。

**macOS / Linux 用户**

在终端进入项目目录，先执行 `chmod +x start.sh` 添加执行权限，再执行 `./start.sh`，程序会自动下载运行环境并启动。启动后默认会自动打开浏览器访问 `http://127.0.0.1:50721`，如果没有自动打开，请手动访问以上网址。余下步骤同上。

> 由于项目主要在 Windows 上开发和测试，macOS 和 Linux 版本可能存在兼容性问题。遇到问题欢迎提交 issue 反馈。~~你都用linux了，给我提个pr不过分吧~~

**资源占用说明**

- 后台常驻内存占用约 50 MB，仅在登录认证时短暂升高，完成后自动释放
- 关闭运行窗口即结束程序，如需长时间监控请开启"开机自启动"
- 如果仅需开机认证一次，建议开启"登录成功后退出"选项，认证成功后程序自动退出，不占用系统资源

---

### 首次配置

首次启动会弹出引导向导，按提示填写即可：

1. **账号** — 填写学号（用户名）、密码、认证地址
2. **运营商** — 选择移动/联通/电信/无，选"自定义"可填写自定义关键词
3. **监控** — 设置检测间隔、暂停时段、是否开机自启
4. **确认** — 检查信息无误后完成

### 日常使用

启动后点击 **启动监控** 按钮即可。程序会自动检测网络状态，断网时自动登录认证，恢复后继续监控。

## 仪表盘

### 运行状态

页面顶部显示四个统计卡片：

- **运行时长** — 本次监控已持续时间
- **检测次数** — 累计网络检测次数
- **失败计数** — 登录失败次数
- **最后检测** — 上次检测的时间

如果正在监控，页面顶部会显示网络状态横幅（已连接/已断开），方便一眼判断当前网络状况。

### 快捷操作

- **手动登录** — 立即执行一次登录认证，不影响自动监控
- **网络测试** — 检测当前网络是否连通
- **开机自启动** — 启用或关闭开机自动运行

### 实时日志

日志区域实时显示程序运行记录，支持：

- 按级别筛选（INFO / WARN / ERROR / DEBUG）
- 关键词搜索
- 自动滚动开关
- 截图预览（登录失败时自动截图，点击可放大查看）

## 设置

设置页面采用标签式布局，分为以下五个标签页：

### 账号设置

| 项目 | 说明 |
|------|------|
| 用户名 | 学号或认证账号 |
| 密码 | 认证密码 |
| 认证地址 | 校园网认证页面的 URL |
| 运营商 | 选择对应的运营商，影响用户名后缀拼接 |

**自定义变量**：可添加自定义变量，在任务配置中通过 `{{变量名}}` 引用。

### 网络与监控

| 项目 | 说明 |
|------|------|
| 检测间隔 | 每隔多少分钟检测一次网络，默认 5 分钟 |
| 网络检测目标 | 用于判断网络是否连通的目标地址，格式 `IP:端口`，多个用逗号分隔 |
| 检测超时 | 网络检测的超时时间（秒） |
| 暂停时段 | 设置暂停开始和结束的小时（0-23），暂停期间不检测网络 |
| 代理 | HTTP 代理地址，留空则不使用代理 |
| 屏蔽系统代理 | 开启后浏览器不走系统代理，避免干扰认证请求 |

### 系统与日志

| 项目 | 说明 |
|------|------|
| 日志级别 | 控制日志详细程度，一般选 INFO 即可 |
| 日志保留天数 | 超过天数的旧日志自动清理 |
| 截图保留天数 | 超过天数的旧截图自动清理 |
| 访问日志 | 记录 HTTP 请求访问日志 |
| 启动后自动监控 | 开启后程序启动即开始监控，无需手动点击 |
| 最小化到系统托盘 | 启动后最小化到系统托盘运行 |
| 启动后自动打开浏览器 | 开启后启动软件时自动打开 Web 控制台页面，默认关闭 |
| 登录成功后退出 | 适用于只需登录一次的场景，登录成功后自动退出程序 |
| 开机自启动 | 查看/设置开机自启状态 |

### 浏览器设置

| 项目 | 说明 |
|------|------|
| 用户代理 | 自定义浏览器 UA 字符串，一般无需修改 |
| 无头模式 | 隐藏浏览器窗口运行，推荐开启 |
| 低资源模式 | 禁止加载图片，减少内存占用 |
| 安全模式 | 使用纯净 Chromium 启动，解决插件冲突问题 |
| 页面超时 | 等待页面加载的最大时间（毫秒），网络慢可适当调大 |
| 浏览器语言 | 浏览器的 Locale 设置，默认 `zh-CN` |
| 时区 | 浏览器时区，默认 `Asia/Shanghai` |
| 视口宽度/高度 | 浏览器窗口的分辨率 |

### 任务设置

查看当前活动任务，快速跳转到任务管理或脚本页面。

### 配置备份

- **创建备份** — 保存当前配置的快照
- **恢复备份** — 还原到某个历史备份（恢复前会自动备份当前配置）
- **导出** — 下载配置文件到本地
- **删除** — 移除不需要的备份

## 任务管理

任务是登录认证的具体执行流程。程序内置了默认任务，一般无需修改。

### 任务列表

每个任务可以：

- **使用** — 设为当前活动任务
- **编辑** — 修改任务配置
- **调试** — 逐步执行任务，查看每一步的截图和结果
- **复制** — 基于现有任务创建副本
- **导出** — 导出为 JSON 文件分享给他人
- **删除** — 移除自定义任务（内置任务不可删除）

支持拖拽排序。还可通过"远程仓库导入"从 [Campus-Auth 任务仓库](https://github.com/Misyra/campus-auth-tasks) 一键导入适配方案，也可点击"分享适配"将自己的任务分享到仓库。

### 新建/编辑任务

- **任务 ID** — 只能包含字母、数字、下划线，创建后不可修改
- **任务名称** — 便于识别的名称
- **描述** — 任务用途说明
- **认证地址** — 留空则使用系统设置中的地址，填写后会覆盖全局地址
- **JSON 配置** — 任务的执行步骤，可点击"加载模板"快速开始。支持的步骤类型包括：输入、点击、选择、等待、截图、休眠、验证码识别（ocr）等。所有操作类步骤支持 `frame` 字段，用于 frameset/iframe 页面中定位子 frame 内的元素

### 调试模式

点击"调试"进入逐步调试：

1. 页面左侧显示浏览器截图，右侧显示任务步骤列表
2. 点击"下一步"逐步执行，每步完成后截图更新
3. 也可以点击"运行剩余"一次性执行完所有步骤
4. 执行结果会显示在每个步骤旁边（成功/失败）

### 导入任务

点击"导入"按钮，选择从他人那里获得的 JSON 文件即可导入任务。

## Python 脚本

Python 脚本通过 HTTP 请求直接登录校园网，无需启动浏览器，资源占用极低。与任务管理（浏览器自动化）互补，适用于登录接口是普通 HTTP 请求的场景。

支持新建、编辑、导入、导出、运行和删除。脚本通过环境变量（`CAMPUS_USERNAME`、`CAMPUS_PASSWORD`、`CAMPUS_ISP`、`CAMPUS_URL`）获取登录参数，`print()` 输出会记录到日志。页面右侧提供了 httpx 和 urllib 两种示例模板。脚本超时默认 60 秒，设为活动任务后自动监控会使用脚本登录。

## 配置方案

配置方案适用于**在多个网络环境间切换**的用户，例如宿舍、教学楼、图书馆使用不同的网关或运营商。每个方案可独立配置账号、认证地址和匹配规则，连接新网络时自动切换，无需手动修改设置。

### 使用步骤

1. **创建方案** — 为宿舍、教室、图书馆等不同网络分别创建配置方案
2. **设置匹配规则** — 填写网关 IP 或 WiFi 名称，系统自动识别当前网络
3. **开启自动切换** — 连接新网络时自动应用对应方案

### 方案编辑

#### 基本信息

- **方案 ID** — 唯一标识，只能包含字母、数字、下划线
- **方案名称** — 便于识别的名称

#### 网络匹配

- **网关 IP** — 当前网络的网关地址（如 `192.168.1.1`）
- **WiFi 名称** — 当前连接的 WiFi SSID

点击"检测当前网络"可自动获取网关 IP 和 WiFi 名称，点击"填入"快速填写。

两项都留空则该方案仅支持手动切换。

#### 账号凭证

- 开启"使用全局账号密码"则使用系统设置中的账号密码
- 关闭后可为本方案配置独立的账号、密码和运营商

#### 认证设置

- **跟随全局认证地址** — 开启使用系统设置中的认证 URL，关闭后可单独配置
- **跟随全局任务** — 开启使用任务管理页面设置的活动任务，关闭后可指定独立任务

#### 高级设置

- **跟随全局设置** — 开启后检测间隔、无头模式、暂停时段等使用全局配置
- 关闭后可为本方案独立设置所有高级选项

### 检测网络

在方案列表页面点击"检测网络"，会显示当前网络的网关 IP 和 WiFi 名称，以及是否匹配到已有方案。

## 常见问题

### Q: 启动后一直显示"网络已断开"

检查认证地址是否正确，确认用户名密码无误。可以先点击"手动登录"测试一次，查看日志中的错误信息。

### Q: 登录失败，提示超时

在设置中调大"页面超时"值（如 30000 毫秒），或关闭"低资源模式"。校园网认证页面加载慢时需要更长的等待时间。

### Q: 检测间隔设多少合适

一般 3-5 分钟即可。设太短会频繁启动浏览器增加资源占用，设太长则断网后恢复较慢。

### Q: 暂停时段有什么用

设置暂停时段后（如凌晨 0-6 点），该时间段内不会检测网络，减少不必要的资源消耗。

### Q: 多个网络环境怎么配置

在"配置方案"页面为每个网络创建独立方案，填写对应的网关 IP 或 WiFi 名称作为匹配规则，开启"自动切换"后程序会自动识别并应用。

### Q: 如何分享任务配置给同学

**本地分享**：在任务管理页面点击"导出"，会下载一个 JSON 文件。同学在他们的程序中点击"导入"选择该文件即可。

**社区分享**：如果你编写了适用于特定校园网的任务，欢迎提交到 [Campus-Auth 任务仓库](https://github.com/Misyra/campus-auth-tasks)，其他用户可以直接从仓库导入。详见 [任务编写指南 - 分享你创建的任务](task-writing-guide.md#分享你创建的任务)。

### Q: 程序占内存太大

在浏览器设置中开启"无头模式"和"低资源模式"。如果只需要登录一次，可以开启"登录成功后退出"功能。
