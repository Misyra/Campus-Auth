# 阶段 3 遗留 P2/P3 修复 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复阶段 3 审查发现的 2 个 P2 + 1 个 P3：(P2a) LoginBridge retry 唤醒延迟最多 5s；(P2b) `check_login_prerequisites` 同步 socket 阻塞 engine loop；(P3) `_wakeup_event` 占位符残留。

**Architecture:** 阶段 3 将 Engine 迁移到 asyncio loop 后，`_engine_loop_async` 用 `await asyncio.wait_for(queue.get(), timeout)` 等待命令。LoginBridge 的 retry 回调在 login executor 线程执行，设置了 `_next_retry_time` 但无法中断 `asyncio.wait_for`——旧代码靠 `_wakeup_event.set()` 立即唤醒，新代码靠 timeout 自然到期，最多延迟 5s。另：`check_login_prerequisites` 仍用同步 `socket.create_connection`，在 engine loop 线程内阻塞。

**Tech Stack:** Python 3.13, asyncio, socket

**Scope note:** 这是整个架构重构的收尾修复，不涉及架构变更，仅补全 A3 未覆盖的路径 + 清理遗留。

## Task 2: check_login_prerequisites 改 async（P2b）

**背景：** `check_login_prerequisites`（decision.py:138）调用 `is_local_network_connected`（probes.py:148）和 `_is_auth_url_reachable`（decision.py:250），两者都用同步 `socket.create_connection`。它们在 `LoginBridge.submit_login`（engine loop 线程）中被调用，阻塞 engine loop 期间所有命令处理停滞。

- `_do_network_check_async`（engine loop async）→ `self._do_async_login` → `self._login_bridge.submit_login`

两条都在 engine loop 内。若 `submit_login` 改 async，调用方需 `await`。

async def _check_interface_connectivity(interface_name: str) -> bool:
    """通过 TCP Connect 验证网卡连通性（async）。"""
    test_targets = [
        ("8.8.8.8", 53),
        ("114.114.114.114", 53),
        ("1.1.1.1", 53),
    ]

    addrs = psutil.net_if_addrs().get(interface_name, [])
    source_ip = None
    for addr in addrs:
        if addr.family == socket.AF_INET:
            source_ip = addr.address
            break

    if not source_ip:
        return False

    local_addr = (source_ip, 0)
    for host, port in test_targets:
        try:
            _, writer = await asyncio.wait_for(
                asyncio.open_connection(host, port, local_addr=local_addr),
                timeout=1.0,
            )
            writer.close()
            await writer.wait_closed()
            return True
        except (OSError, TimeoutError):
            continue

    return False

```python
async def _is_auth_url_reachable(
    auth_url: str,
    extra_targets: Sequence[str] | None = None,
    source_ip: str | None = None,
) -> bool:
    """检查认证地址及附加目标的 TCP 可达性（async）。"""
    if not auth_url and not extra_targets:
        return True

    local_addr = (source_ip, 0) if source_ip else None

    async def _check_host_port(host: str, port: int, label: str) -> bool:
        try:
            _, writer = await asyncio.wait_for(
                asyncio.open_connection(host, port, local_addr=local_addr),
                timeout=3,
            )
            writer.close()
            await writer.wait_closed()
            logger.debug("认证可达性检测通过: {}", label)
            return True
        except (OSError, TimeoutError) as exc:
            logger.debug("认证可达性检测失败: {} -- {}", label, exc)
            return False

    if extra_targets:
        from app.network.parsers import parse_host_port

        targets = parse_host_port(list(extra_targets))
        if targets:
            for host, port in targets:
                if await _check_host_port(host, port, f"{host}:{port}"):
                    return True
        logger.debug("自定义检测目标均不可达")
        return False

    if auth_url:
        try:
            parsed = urlparse(auth_url)
            host = parsed.hostname
            if not host:
                logger.debug("认证地址 hostname 解析失败，视为不可达: {}", auth_url)
                return False
            port = parsed.port or (443 if parsed.scheme == "https" else 80)
            if await _check_host_port(host, port, auth_url):
                return True
        except Exception as exc:
            logger.debug("认证地址可达性检测异常: {} -- {}", auth_url, exc)
            return False

    return False
```

**2. 风险点:**
- **Task 2 submit_login → async 级联**：`submit_login` 改 async 后，`_do_async_login`、`_handle_login`、`_process_command_async` 都需 await。调用链：`_engine_loop_async` → `_process_command_async` → `_handle_login` → `submit_login` → `check_login_prerequisites` → `is_local_network_connected` / `_is_auth_url_reachable`。全链路在 engine loop 内，可安全 await。
- **LoginBridge.submit_login 的 on_complete 回调**：`on_complete` 在 `_on_done`（done_callback）中调用，运行在 login executor 线程。submit_login 改 async 不影响 on_complete 的线程语义——on_complete 仍由 Future.done_callback 触发。
- **is_local_network_connected 的其他调用点**：grep 确认仅 `decision.py:154` 一处调用。若 other 模块有同步调用，需保留同步兼容版本。
- **NOOP 命令的 response_future**：NOOP 命令不设置 response_future（`None`），`_process_command_async` 的 except/finally 分支检查 `cmd.response_future and not cmd.response_future.done()` 安全跳过。
- **_check_interface_connectivity 的 psutil 调用**：`psutil.net_if_addrs()` 仍是同步调用（非 socket），在 async 函数内可接受（快速返回，不阻塞 loop）。
