## 关键问题与不一致

| ID | 类别 | 位置 | 说明 | 影响 |
| --- | --- | --- | --- | --- |
| BUG-01 | 逻辑错误 | `app.py:201-210` | `login_then_exit` 模式使用 `load_ui_config()`（全局配置）而非 `load_runtime_config()`（活动方案），导致方案独立账号/认证地址不会生效。 | 使用非全局方案时可能登录失败或登录到错误门户。 |
| BUG-02 | 配置失效 | `app.py:214-229` + `backend/config_service.py:344-349` | `retry_interval` 从 `runtime_config.get("retry_interval")` 读取，但运行时配置仅在 `retry_settings.retry_interval` 中存放该值。 | 用户设置的重试间隔被忽略，实际固定为默认值 5 秒。 |
| BUG-03 | 数据丢失 | `src/task_executor.py:220-238` + `backend/task_service.py:52-56` | `TaskConfig.to_dict()` 未输出 `step_delay`，而 `GET /api/tasks/{id}` 直接返回 `to_dict()` 结果。 | 任务包含 `step_delay` 时，经过读取/编辑/保存会丢失该字段。文档仍宣称支持该字段。 |
| BUG-04 | 约束冲突 | `backend/schemas.py:12-55` + `backend/schemas.py:226-245` | `MonitorConfigPayload.max_retries` 限制为 `1~5`，但 `SystemSettings.max_retries` 允许 `0~10`。若 `settings.json` 存在 `0` 或 `>5`，`load_ui_config/load_runtime_config` 构建 payload 可能触发校验错误。 | 配置读取可能失败，导致 `/api/config` 等接口报错。 |
| DOC-01 | 文档不一致 | `README.md:197` vs `app.py:101-103` | README 使用环境变量 `AUTO_OPEN_BROWSER`，但实际读取的是 `CAMPUS_AUTH_AUTO_OPEN_BROWSER`（并由自启动脚本设置）。 | 按 README 配置将不生效，造成用户误解。 |
| DOC-02 | 文档不一致 | `README.md:148-153` vs `settings.json:7-8` | README 声明 `LOGIN_URL` 默认 `http://172.29.0.2`，但仓库内默认配置为 `http://127.0.0.1:5500/ds.html`。 | 源码运行时默认行为与文档不一致，容易导致误判。 |
| DOC-03 | 文档不一致 | `README.md:168` + `doc/task-manual.md:465` vs `src/utils/config.py` | 文档列出 `MAX_CONSECUTIVE_FAILURES` 环境变量，但代码未读取该变量，实际使用的是 `RETRY_MAX_RETRIES`。 | 用户按文档配置不会生效。 |
| DOC-04 | 文档不一致 | `doc/task-manual.md:344-439` vs `backend/main.py` | API 文档未列出已实现的接口：`/api/safe-mode`、`/api/debug/*`、`/api/repo/fetch`、`/api/repo/task`、`/api/docs/task-writing-guide`。 | 文档与实际 API 能力不一致。 |
| DOC-05 | 文档不一致 | `README.md:579-580` + `frontend/partials/pages/settings.html:169-178` vs `app.py:215-216` | 文档/前端提示登录成功后退出模式“最多 3 次重试”，但代码实际使用 `max_retries`（允许 5 或手动配置更大）。 | 用户预期与实际行为不一致。 |
